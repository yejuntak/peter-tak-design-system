A production-grade design system,
optimized for humans and AI.

Atomic-design organized. Plain CSS, no preprocessor. Machine-readable tokens (W3C DTCG-ish JSON). Component specs written so AI agents can produce on-system markup without seeing the rendered output. The whole system is six CSS files and a folder of markdown specs.

Tokens

tokens/color.json Color Three-tier: primitive grays + amber, semantic aliases (bg, ink, accent), component-level (—ink-soft, —surface-2). tokens/typography.json Typography Aeonik display + Geist Mono. Scale from clamp(40, 5vw, 64) for hero down to 10px tiny. tokens/spacing.json Spacing Geometric scale 4 → 128. Pick the closest token; don't split the difference. tokens/radius.json Radius r-0 (kbd) → r-full (pill). Pick by role, not size. tokens/motion.json Motion Two durations × two easings. Anything else needs justification.

New components (Vita Health case)

.m-lab-result Lab result Health value with range + trend + plain-language explanation. Never a raw number. .m-progress-bar Progress bar Adherence as progress, never compliance. API enforces the framing. .m-confidence-tier Confidence tier High · Medium · Low — language wrappers around model scores. Never expose the raw number. .m-action-trio Action trio Accept · Modify · Dismiss. Modify is first-class — partial acceptance is the common case. .o-recommendation-card Recommendation card Composes tier + direction-not-directive title + rationale + sources + action-trio.

Reference screens

screens/vita-health/ Vita Health · Patient journey 5 reference screens: onboarding, lab interpretation, care plan, progress framing, ML recommendation. Built entirely from this system.
