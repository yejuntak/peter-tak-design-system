Vita Health.
Patient journey, built from this design system.

Five reference screens reconstructing the core patient-journey patterns from a 3-year UX Lead engagement at Onlife Health / Pager Health (Vita-branded preventative care app, 2019–2022). Every screen is built from the Peter Tak Design System with no page-scoped CSS. The components that made these screens possible — .m-lab-result, .m-confidence-tier, .m-action-trio, .m-progress-bar — are documented as first-class system components.

New system components this case added

Building these screens surfaced eight first-class components that were missing from the base system. Each is documented in components/ with anatomy, rules, when-to-use, and HTML examples.

.m-lab-result Lab value with range + trend + explanation .m-range-indicator Visual position on a value spectrum .m-trend-arrow Direction vs. previous reading .m-progress-bar Progress framing, never compliance .m-confidence-tier High · Medium · Low (language, not scores) .m-action-trio Accept · Modify · Dismiss .o-recommendation-card Composes tier + body + action-trio .o-care-plan Direction → patient-authored plan .o-health-snapshot Curated multi-result grid
